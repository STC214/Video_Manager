# Go + Win32 UI 卡死风险与视觉风格规范

## 目标

本文档记录当前已知可能导致 Go + Win32 桌面程序 UI 卡死、假死或明显掉帧的风险点，并定义首版 UI 的主题色风格。

风险范围不只覆盖当前视频归档功能，也覆盖后续扩展桌面程序功能时常见的 Win32 / Windows API / 文件系统 / Shell / COM / 外部进程相关卡死点。即使首版暂时不用到某些能力，也应在架构上避免把这些风险引入 UI 线程。

实现时应把 UI 线程视为只负责消息循环、控件状态更新和轻量绘制的线程。所有耗时文件操作、网络路径访问、批量计算和大量日志写入都必须离开 UI 线程执行。

## UI 主题风格

整体风格：深色、时髦、克制、偏工具型。

设计目标：

- 深色背景，不使用默认白底窗口。
- 标题栏不能使用系统默认白色标题栏。
- 主界面应保持专业、清晰、低干扰，适合长时间查看任务进度。
- 按钮、输入框、列表、日志区域需要和深色主题一致。
- 重点状态使用少量高亮色，不使用大面积高饱和渐变。

建议色板：

```text
窗口背景：#101418
面板背景：#171D23
输入背景：#0D1116
边框颜色：#2A333D
主文字：#E6EDF3
次级文字：#9AA7B2
禁用文字：#5C6670
强调色：#4DA3FF
成功色：#43C07A
警告色：#E6A23C
错误色：#F56C6C
```

标题栏要求：

- 标题栏必须适配深色主题，不能保留默认白色标题栏。
- 优先使用 Windows 支持的沉浸式深色标题栏属性。
- 如果系统版本不支持深色标题栏，应使用自绘标题栏或在文档中明确降级表现。
- 自绘标题栏必须保留最小化、最大化、关闭、拖拽、双击最大化和窗口缩放等基础行为。

## 总体线程原则

- UI 线程只处理 Win32 消息循环、控件事件和轻量状态渲染。
- 扫描、dry-run 生成、正式移动、清理、日志落盘、TSV 导出都必须在后台 goroutine 或工作线程执行。
- 后台任务不能直接操作 Win32 控件句柄，应通过消息、队列或主线程调度更新 UI。
- UI 更新需要节流，避免每处理一个文件就立即刷新大量控件。
- 所有长任务都必须可取消，并能把取消状态反馈到 UI。

## 已知卡死风险

### 1. 递归扫描目录

风险：

- 源目录中文件数量巨大。
- 原目录层级很深。
- 网络 SMB 目录枚举慢。
- 某些目录权限不足，访问时阻塞或等待超时。

要求：

- 递归扫描必须在后台执行。
- 扫描期间 UI 显示进度、当前路径和可取消状态。
- UI 不应等待整个扫描完成才刷新统计。
- 遇到不可访问目录应记录错误并继续。
- 对 SMB / UNC 路径，目录枚举可能很慢，不能阻塞 UI 线程。

### 2. 视频扩展名统计和大小统计

风险：

- 对每个文件调用 `Stat` 读取大小和时间。
- SMB 设备返回文件属性慢。
- 大量文件统计后一次性刷新 UI。

要求：

- 文件属性读取在后台执行。
- 统计结果批量合并后节流刷新 UI。
- 可以按固定间隔刷新，例如每 100ms 到 250ms 刷新一次。

### 3. Dry-run 计划生成

风险：

- 对几千到几十万个文件排序。
- 生成大量目标路径。
- 检查目标重名时频繁访问磁盘。
- 构建结构预览树时一次性创建过多 UI 节点。

要求：

- 排序和路径计算在后台执行。
- 重名检查应缓存目标叶目录内已有文件名，避免重复枚举同一目录。
- 结构预览先生成数据模型，再按需渲染。
- UI 列表使用虚拟列表或分页，不要一次性插入全部行。

### 4. 正式移动文件

风险：

- 大文件移动、跨盘复制或网络复制耗时很长。
- SMB 路径可能断开、超时或权限变化。
- copy + verify + delete 会长时间占用任务。
- 直接在 UI 线程移动会导致窗口无响应。

要求：

- 移动必须在后台执行。
- 每个文件移动完成后写入任务状态，但 UI 更新要节流。
- 对网络路径失败项记录错误并继续。
- 取消任务时，当前文件操作完成或失败后停止继续处理。
- 不应在文件复制过程中阻塞消息循环。

### 5. 空目录清理

风险：

- 自底向上扫描大量目录。
- 判断隐藏文件、非空目录和目标结构保护时需要访问目录内容。
- SMB 目录删除或枚举可能很慢。

要求：

- 清理预览和正式清理都在后台执行。
- 删除前再次确认目录为空。
- 清理阶段的 UI 只显示摘要和最近处理路径。

### 6. 日志和运行清单写入

风险：

- 每个文件都同步写日志。
- 日志区域每追加一行就立即刷新。
- 运行清单写入失败阻塞主流程。

要求：

- 日志落盘使用缓冲写入。
- UI 日志区域限流刷新。
- 日志控件保留最近 N 行，完整日志写入文件。
- 运行清单写入失败应显示错误，但不能卡死 UI。

### 7. 大型列表控件渲染

风险：

- dry-run 可能包含数千到数十万行。
- Win32 ListView 一次性插入大量行会明显卡顿。
- 每行更新状态会触发大量重绘。

要求：

- 预览列表优先使用虚拟列表。
- 大量状态变更应批量刷新。
- 移动中只高频更新摘要、进度条和最近处理项。
- 完整列表可以低频刷新或在任务完成后刷新。

### 8. 进度条和状态文本刷新过于频繁

风险：

- 后台任务每处理一个文件就发送 UI 消息。
- 文件很多时 UI 消息队列堆积。

要求：

- 进度刷新节流，例如每 100ms 到 250ms 或每处理一定数量文件刷新一次。
- 后台任务发送最新状态即可，允许覆盖旧状态。
- UI 不需要展示每个微小步骤。

### 9. Win32 消息循环阻塞

风险：

- 在窗口过程里执行耗时操作。
- 在按钮点击事件里直接扫描或移动。
- 同步等待后台 goroutine 完成。
- 使用阻塞 channel 接收导致窗口过程卡住。

要求：

- 窗口过程只做轻量分发。
- 按钮事件只启动后台任务并立即返回。
- UI 线程不能同步等待后台任务。
- 后台任务通过 `PostMessage` 或线程安全队列通知 UI。
- Go 主 UI goroutine 必须使用 `runtime.LockOSThread()` 固定到创建窗口的系统线程；Win32 消息队列属于线程，迁移后可能造成窗口启动后立即无人处理消息。

### 10. 跨 goroutine 操作 UI 控件

风险：

- 后台 goroutine 直接调用 Win32 控件 API。
- 造成竞态、崩溃或偶发卡死。

要求：

- 所有控件更新都回到 UI 线程执行。
- 后台任务只提交状态对象，不持有控件更新逻辑。
- UI 状态模型和任务状态模型要分离。

### 11. 对话框和确认框阻塞

风险：

- 移动过程中弹出模态错误框。
- 大量错误连续弹窗。
- 网络路径错误导致对话框阻塞任务。

要求：

- 长任务中不弹出逐文件模态框。
- 错误写入日志和错误列表。
- 任务结束后统一展示摘要。
- 只有开始前的危险操作确认可以使用模态对话框。

### 12. 路径过长或特殊字符

风险：

- Windows 传统路径长度限制导致文件操作卡住或失败。
- 中文、空格、特殊标点在日志、TSV 和控件中显示异常。

要求：

- 路径处理集中封装。
- 长路径失败应快速记录错误，不反复重试卡住 UI。
- UI 和 TSV 均使用 UTF-8 或明确编码策略。

### 13. SMB / UNC 网络路径

风险：

- 软路由或 NAS 响应慢。
- 网络共享登录状态失效。
- 文件枚举、复制、删除操作阻塞时间不可控。
- 某些 SMB 服务对 rename、mtime、隐藏属性支持不完整。

要求：

- 所有 SMB / UNC 文件访问都在后台任务中执行。
- 正式移动前重新检查源文件可访问性。
- direct move 失败时退化为 copy + verify + delete。
- 网络错误只影响当前文件或目录，不应让 UI 卡死。
- UI 应显示“网络路径访问较慢”一类的状态提示，而不是无响应。

### 14. 取消任务

风险：

- UI 点击取消后后台任务不响应。
- 当前复制操作无法立即停止，用户误以为程序卡死。

要求：

- 取消按钮应立即更新为“正在取消”状态。
- 后台任务在文件边界检查取消信号。
- 当前文件操作无法立即中断时，应在 UI 上说明正在等待当前文件操作结束。
- 取消后保留运行清单和已完成结果。

### 15. 文件选择对话框和 Shell 对话框

风险：

- `GetOpenFileName`、`SHBrowseForFolder`、`IFileDialog` 等对话框可能枚举慢速网络位置。
- 用户最近访问位置指向失效的 SMB / NAS 路径。
- Shell 扩展或第三方右键菜单扩展响应慢。

要求：

- 打开对话框前 UI 应处于明确空闲或可等待状态。
- 对话框返回后再次验证路径可访问性。
- 不在后台任务进行中弹出文件选择对话框。
- 如果需要更复杂的目录选择，优先使用可控的异步路径输入和验证流程。

### 16. Shell 图标、缩略图和文件属性

风险：

- 读取文件图标、缩略图、媒体属性可能触发 Shell 扩展。
- 视频缩略图生成可能很慢，甚至被损坏文件卡住。
- 网络文件图标和属性读取可能访问远端。

要求：

- 首版不生成视频缩略图。
- 文件图标和缩略图读取必须后台化并可取消。
- UI 列表先显示通用图标，异步补充图标。
- 对慢速或失败的缩略图请求设置超时或跳过策略。

### 17. COM 初始化和调用

风险：

- Shell、文件对话框、任务栏进度、拖放等能力可能依赖 COM。
- 在错误线程模型中调用 COM 可能卡死或行为异常。
- UI 线程 STA 与后台 MTA 混用不当会导致同步等待。

要求：

- 明确每个线程的 COM 初始化方式。
- UI 线程如需 Shell UI 能力，应使用 STA 并保持消息循环畅通。
- 后台线程避免同步等待 UI 线程 COM 回调。
- COM 对象释放和线程退出顺序要明确。

### 18. 拖放文件和剪贴板

风险：

- 拖入大量文件或目录时，解析拖放数据耗时。
- 剪贴板可能由其他进程提供数据，读取时发生阻塞。
- 网络路径拖放可能触发 Shell 查询。

要求：

- 拖放事件只收集最小必要数据并快速返回。
- 大量路径解析放到后台任务。
- 剪贴板读取只在用户明确触发时进行，并避免长时间阻塞 UI。

### 19. 外部进程调用

风险：

- 后续可能调用 `ffprobe`、压缩工具、播放器、资源管理器或脚本。
- 同步等待外部进程会让 UI 卡死。
- 子进程 stdout / stderr 未读取可能导致子进程阻塞。

要求：

- 外部进程必须后台启动和等待。
- stdout / stderr 必须异步读取或重定向。
- UI 只显示摘要、进度和最近输出。
- 子进程超时、取消和退出码都要记录。

### 20. 数据库、索引和缓存

风险：

- 后续增加数据库索引、历史记录、任务恢复或 hash 缓存时，查询和写入可能阻塞。
- SQLite 大事务、锁等待或网络盘数据库文件可能导致 UI 假死。

要求：

- 数据库操作不在 UI 线程执行。
- 大批量写入使用事务，但事务执行在后台。
- UI 查询使用分页或异步查询。
- 数据库文件优先放在本地应用数据目录，不建议放在 SMB 网络盘。

### 21. Hash、去重和媒体解析

风险：

- 文件 hash 需要完整读取大文件。
- 去重需要大量 I/O 和 CPU。
- 媒体解析可能遇到损坏文件、超长读取或第三方库阻塞。

要求：

- hash 和媒体解析必须作为独立后台任务。
- 支持暂停、取消和限速。
- UI 显示阶段性进度，不阻塞其他基础操作。
- 对单文件解析设置失败和跳过策略。

### 22. 杀毒软件、索引服务和文件锁

风险：

- 新创建或复制的大文件可能被杀毒软件扫描。
- Windows Search 或其他索引服务可能短暂锁定文件。
- 文件锁导致重试时 UI 长时间无响应。

要求：

- 文件锁错误应记录并按有限次数后台重试。
- 重试需要退避等待，不能阻塞 UI。
- UI 应显示“等待文件释放”而不是无响应。
- 超过重试次数后标记失败并继续。

### 23. GDI / USER 对象泄漏

风险：

- 自绘深色 UI、字体、画刷、位图、图标创建后未释放。
- 长时间运行后 GDI / USER 对象耗尽，窗口绘制异常或卡死。

要求：

- 所有 GDI 对象创建和释放必须成对管理。
- 主题画刷、字体、图标应集中缓存和销毁。
- 避免在 `WM_PAINT` 中反复创建对象。
- 调试阶段定期观察进程 GDI / USER 对象数量。

### 24. WM_PAINT 和自绘控件

风险：

- `WM_PAINT` 中执行复杂计算、文件读取或布局重算。
- 自绘标题栏、深色控件和列表过度重绘。
- 未正确处理无效区域导致持续重绘。

要求：

- `WM_PAINT` 只做绘制，不做 I/O 和重计算。
- 布局和数据准备在绘制前完成。
- 大面积刷新要合并无效区域。
- 自绘控件需要双缓冲，避免闪烁和高 CPU。

### 25. DPI、窗口缩放和布局计算

风险：

- DPI 变化时同步重建大量控件。
- 窗口 resize 过程中频繁重排复杂布局。
- 字体和控件尺寸重复计算导致拖动窗口卡顿。

要求：

- DPI 变化时缓存字体和尺寸。
- `WM_SIZE` 处理要轻量，只做必要布局。
- 大型列表和日志区域不在每次 resize 时重建数据。

### 26. 定时器和轮询

风险：

- 使用高频 Win32 timer 或 goroutine ticker 轮询状态。
- 多个定时器同时刷新 UI。
- 窗口最小化或后台时仍高频刷新。

要求：

- UI 定时刷新频率保持低频且可控。
- 后台任务用事件推送为主，定时轮询为辅。
- 窗口最小化时降低刷新频率。
- 关闭窗口时停止所有 timer / ticker。

### 27. goroutine 泄漏和 channel 堵塞

风险：

- 每次扫描或移动启动 goroutine，但任务结束后未退出。
- 无缓冲 channel 发送 UI 状态时没人接收。
- 关闭窗口后后台 goroutine 继续发送消息。

要求：

- 每个任务有明确生命周期和取消上下文。
- UI 状态通道使用非阻塞发送或可覆盖的最新快照。
- 窗口关闭时通知所有后台任务停止。
- 任务结束后释放 channel、timer 和文件句柄。

### 28. 锁顺序和死锁

风险：

- UI 线程持有状态锁时等待后台任务。
- 后台任务持有状态锁时发送同步 UI 消息。
- 日志、任务状态、配置状态多把锁顺序不一致。

要求：

- UI 线程不持锁执行耗时操作。
- 后台任务不在持锁状态下调用 UI 通知。
- 锁顺序需要固定，复杂状态优先用单线程事件循环或消息队列管理。

### 29. 同步 SendMessage 和跨线程消息

风险：

- 后台线程使用 `SendMessage` 调用 UI，UI 正在等待后台线程，形成死锁。
- 向错误窗口句柄发送消息导致阻塞或崩溃。

要求：

- 后台通知 UI 优先使用 `PostMessage` 或线程安全队列。
- 避免后台线程对 UI 使用同步 `SendMessage`。
- 窗口销毁后停止投递或校验窗口句柄有效性。

### 30. 配置文件、主题文件和字体加载

风险：

- 启动时同步读取大型配置或坏配置。
- 配置文件位于网络盘。
- 字体加载或主题资源读取失败导致 UI 初始化卡住。

要求：

- 启动路径只读取最小配置。
- 大型配置和历史记录延迟加载。
- 配置解析失败要快速回退默认值。
- 主题资源放在本地程序目录或内嵌资源中。

### 31. 系统电源、休眠和网络恢复

风险：

- 长任务期间系统休眠。
- 唤醒后 SMB 连接失效。
- 文件句柄、网络路径或任务状态过期。

要求：

- 恢复后重新验证当前源路径和目标路径。
- 对过期句柄或网络错误记录失败并继续。
- UI 显示“系统恢复后正在重新检查路径”。

### 32. 高 CPU 计算和排序

风险：

- 大量文件排序、分组、统计、hash 或索引构建占满 CPU。
- Go 后台 goroutine 过多导致 UI 线程抢不到调度。

要求：

- CPU 密集任务限制并发。
- 大型排序和统计任务可分阶段处理。
- UI 线程保持独立且持续处理消息。
- 任务进度更新按阶段汇报，不刷屏。

## 推荐 UI 状态模型

建议 UI 只订阅任务快照：

```go
type TaskSnapshot struct {
    State          string
    TotalFiles     int
    PlannedFiles   int
    DoneFiles      int
    FailedFiles    int
    ConflictFiles  int
    CurrentPath    string
    LastError      string
    UpdatedAt      time.Time
}
```

后台任务定期发布 `TaskSnapshot`，UI 线程按节流策略读取最新快照并刷新控件。

## 实现检查清单

- 扫描不在 UI 线程执行。
- dry-run 生成不在 UI 线程执行。
- 移动和复制不在 UI 线程执行。
- 空目录清理不在 UI 线程执行。
- 日志落盘不在 UI 线程执行。
- 文件对话框和 Shell 调用不在长任务中随意弹出。
- 图标、缩略图、媒体属性读取不阻塞 UI。
- 外部进程启动、等待和输出读取不阻塞 UI。
- 数据库、hash、索引和媒体解析不阻塞 UI。
- ListView 使用虚拟列表或分页。
- UI 刷新有节流。
- 后台任务不直接操作控件。
- 后台任务优先使用 `PostMessage` 或队列通知 UI，避免跨线程 `SendMessage`。
- 错误不逐文件弹模态框。
- 取消按钮能立即反馈。
- SMB / UNC 慢响应不会让窗口无响应。
- GDI / USER 对象无持续泄漏。
- `WM_PAINT` 不做 I/O、复杂计算或大量对象创建。
- goroutine、timer、channel 和文件句柄能随任务结束释放。
- 标题栏适配深色主题，不使用默认白色标题栏。
- 所有主要控件颜色与深色主题一致。

## 当前项目审查状态

已覆盖：

- 扫描、dry-run 生成、正式移动、撤销和正式空目录清理都已放到后台 goroutine。
- 扫描、dry-run、移动、撤销均可通过取消按钮发出取消信号。
- 后台任务通过 `PostMessage` 通知 UI，未直接跨 goroutine 操作控件。
- 扫描使用系统原生不确定进度条，移动和撤销使用确定进度条。
- 进度条 range 只在总数变化时更新，并使用 `WS_CLIPCHILDREN` 降低子控件闪动。
- UI 日志和移动进度有节流，日志控件只保留最近 200 行。
- 完整日志已异步写入程序目录 `data/logs/`，避免 UI 日志截断后丢历史。
- 启动阶段默认不创建日志文件；任务日志首次使用时懒加载。启动轨迹仅在 `VIDEO_MANAGER_STARTUP_TRACE=1` 时通过后台单写入队列落盘。
- Dry-run 预览已分页显示，完整明细通过 TSV 保留。
- 源目录可读性、目标目录可访问性和目录名非法字符已增加前置校验。
- 目录选择已改用资源管理器风格的 `IFileOpenDialog`，通过 `IShellItem` 获取文件系统路径；仅在 COM 创建失败时回退旧选择器。
- 移动链路使用长路径封装、有限 I/O 重试、直接 move 失败后 copy + verify + delete。
- UNC 路径和 Windows 识别为 remote drive 的映射盘会使用更宽容的网络退避重试，最长可等待数十秒后再判定当前操作失败。
- 大文件复制已改为分块复制，并在块边界检查取消信号。
- 窗口销毁时会向扫描、dry-run 和移动任务发出取消信号。
- GDI 画刷和字体集中创建，并在退出时释放。
- Win32 UI 生命周期已使用 `runtime.LockOSThread()` 固定在同一系统线程，避免窗口创建线程与消息循环线程分离导致随机假死。
- UI 状态对象在消息循环结束前通过 `runtime.KeepAlive` 保持存活，避免仅由 Win32 用户数据指针引用时被提前回收。
- Dry-run 目标冲突检查按叶目录缓存目录项，避免 SMB 路径逐文件 `Stat` 造成大量远程往返。
- 所有操作按钮句柄已纳入状态表；任务期间路径和容量配置会禁用，只保留取消操作。
- 进度条与计算区不再重叠，输出框使用多行滚动控件，按钮使用统一宽度和深色自绘样式。
- 输入框改用自身 `WS_BORDER` 绘制完整四边框，避免深色主题下 `WS_EX_CLIENTEDGE` 左侧阴影边不可见。
- 主窗口和子控件在隐藏状态完成创建，设置字体时不逐控件重绘，最后通过一次 `ShowWindow` 完成首帧显示。
- 配置加载、任务结束保存和关闭前保存均在后台执行；UI 线程只生成配置快照，后台保存完成后再销毁窗口。
- 任务运行中关闭窗口时先发送取消并继续处理消息，等待后台任务真正结束后再保存配置和退出，避免进程退出截断部分复制文件的清理。

残余限制：

- 资源管理器式 `IFileOpenDialog` 本身仍可能因失效 SMB 最近位置或第三方 Shell 扩展短暂卡住。
- 底层 SMB 读写或目录枚举可能在单次 Windows 系统调用内阻塞，程序无法保证硬超时中断。
- 关闭窗口后后台任务可能仍在当前系统调用结束后才退出；这是 Windows 文件 I/O 的常见限制，需要用状态提示和任务边界取消配合处理。
