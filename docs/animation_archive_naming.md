# 动漫视频归档命名规范

## 目标

将零散视频文件整理为两层目录结构，使每个叶目录下的视频数量接近，方便浏览、备份和后续迁移。

该规范不按原始目录数量整理，也不按日期整理，只追求归档后的叶目录容量相对均衡。

## 目录结构

采用两层结构：

```text
Animation/
  Season_001/
    Episode_001/
    Episode_002/
    Episode_003/
    Episode_004/
  Season_002/
    Episode_005/
    Episode_006/
    Episode_007/
    Episode_008/
  Season_003/
    Episode_009/
    Episode_010/
    Episode_011/
    Episode_012/
```

## 命名规则

第一层目录使用：

```text
Season_001
Season_002
Season_003
...
```

含义：归档批次的大分组。

`Season` 默认使用三位数编号，和三层模式保持一致，方便后续工具共用同一套命名逻辑。

第二层目录使用：

```text
Episode_001
Episode_002
Episode_003
...
```

含义：实际存放视频的叶目录。

## 分组规则

- 不按原始目录名整理。
- 不按日期整理。
- 按最后修改时间从早到晚排列文件，越早的文件进入编号越小的叶目录；时间相同时按原相对路径和文件名排序。
- 每个 `Episode_xxx` 下最多放 25 个视频。
- 只有最后一个 `Episode` 可能少于 25 个视频。
- 每 4 个 `Episode` 归入一个 `Season`。

示例：

```text
Episode_001: 第 1-25 个视频
Episode_002: 第 26-50 个视频
Episode_003: 第 51-75 个视频
Episode_004: 第 76-100 个视频
Season_001 结束

Episode_005: 第 101-125 个视频
Episode_006: 第 126-150 个视频
...
```

## 文件命名

- 默认保留原文件名。
- 不主动改标题。
- 不按内容重命名。
- 如果目标目录中出现重名文件，追加编号避免覆盖：

```text
原文件名.mp4
原文件名_dup001.mp4
原文件名_dup002.mp4
```

## 清理规则

移动完成后：

1. 检查原目录是否已经为空。
2. 只删除空目录。
3. 不删除非空目录。
4. 不删除含隐藏文件的目录。
5. 不删除新建目标结构。
6. 不处理非视频文件，除非另行确认。

## 推荐视频扩展名范围

```text
.mp4
.mkv
.avi
.mov
.wmv
.flv
.webm
.m4v
.ts
.m2ts
.mpg
.mpeg
.rmvb
.3gp
```
